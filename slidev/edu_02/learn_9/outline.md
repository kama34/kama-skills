# Воркшоп: Практический MLOps
## Слайд 1: Обложка
- Заголовок: «MLOps: от ноутбука до продакшена за 1 день»
- Подзаголовок: Практический воркшоп для ML-инженеров
- Спикер: Алексей Козлов, Lead ML Engineer

## Слайд 2: 87% ML-моделей не доходят до продакшена
- Исследование Gartner, 2025
- Главная причина: отсутствие инфраструктуры
- Средний срок от прототипа до деплоя: 4,5 месяца

## Слайд 3: Секция — Что такое MLOps и зачем он нужен

## Слайд 4: MLOps = DevOps + Data Engineering + ML
- Автоматизация обучения и деплоя моделей
- Мониторинг дрейфа данных и качества
- Версионирование моделей и экспериментов
- Воспроизводимость результатов

## Слайд 5: Уровни зрелости MLOps
- Level 0: ручной процесс, jupyter-notebooks
- Level 1: автоматизация пайплайна обучения
- Level 2: CI/CD для ML, автоматический ретрейнинг
- Level 3: полная автоматизация + A/B тестирование

## Слайд 6: Секция — Стек инструментов

## Слайд 7: Минимальный стек для старта
- Эксперименты: MLflow / Weights & Biases
- Оркестрация: Airflow / Prefect
- Модельный реестр: MLflow Model Registry
- Деплой: Seldon Core / BentoML

## Слайд 8: Продвинутый стек для масштаба
- Feature Store: Feast / Tecton
- Мониторинг: Evidently AI / WhyLabs
- Инфраструктура: Kubernetes + Kubeflow
- Данные: DVC + Delta Lake

## Слайд 9: Секция — Практика: деплоим модель

## Слайд 10: Пайплайн за 4 шага
- Шаг 1: Обучение модели + логирование в MLflow
- Шаг 2: Регистрация в Model Registry
- Шаг 3: Упаковка в Docker-контейнер
- Шаг 4: Деплой в Kubernetes через Seldon

## Слайд 11: Мониторинг — страховка от деградации
- Data drift: статистические тесты каждый час
- Model performance: метрики на живом трафике
- Алерты: PagerDuty при падении accuracy > 5%
- Автоматический rollback при критических падениях

## Слайд 12: Секция — Типичные ошибки

## Слайд 13: 5 ошибок, которые мы допустили
- Обучение на всех данных без версионирования
- Отсутствие staging-среды для моделей
- Игнорирование data drift первые 3 месяца
- Ручной деплой «пока работает»

## Слайд 14: Без MLOps каждое обновление модели — рулетка
- Непредсказуемое поведение в продакшене
- Невозможность откатиться к предыдущей версии
- Часы на ручной деплой вместо минут

## Слайд 15: Чеклист для самопроверки
- Есть ли версионирование данных?
- Можно ли воспроизвести эксперимент?
- Есть ли мониторинг модели в продакшене?
- Автоматизирован ли деплой?
