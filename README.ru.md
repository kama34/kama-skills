# Kama Skills (Claude Code Skills)

[![en](https://img.shields.io/badge/lang-English-blue)](README.md) [![ru](https://img.shields.io/badge/lang-Русский-green)](README.ru.md)

Коллекция **скиллов** для [Claude Code](https://docs.anthropic.com/en/docs/claude-code) — подключаемых модулей-инструкций, которые дают Claude новые возможности. Открыто для совместного редактирования и улучшения.

> Скилл — это markdown-файл, который учит Claude Code делать то, что он не умеет из коробки: генерировать презентации, собирать проекты, запускать сложные пайплайны и многое другое.

## Доступные скиллы

| Скилл | Описание | Сложность |
|-------|----------|-----------|
| **[slidev](./skills/slidev/)** | Генерация готовых Slidev-презентаций из текстовых аутлайнов. Пресеты, автоматический подбор изображений, визуальный QA, экспорт в PDF/PNG, режим самообучения. | Продвинутый |

## Что такое скиллы?

Скиллы — это markdown-файлы с инструкциями, которые лежат в `.claude/skills/` внутри вашего проекта. При вызове Claude читает скилл и точно следует его инструкциям — как эксперт, получивший детальный регламент.

В отличие от обычных промптов, скиллы могут:
- Описывать многошаговые процедуры с ветвлениями
- Ссылаться на вспомогательные файлы (гайды по дизайну, синтаксису, примеры)
- Содержать переиспользуемые шаблоны кода
- Предоставлять подкоманды (`--edit`, `--export`, `--dev`, `--learn`)
- Самоулучшаться через циклы обратной связи

## Быстрый старт

### 1. Скопируйте скилл в свой проект

```bash
# Создайте директорию для скиллов
mkdir -p .claude/skills

# Скопируйте нужный скилл (например, slidev)
cp -r path/to/claude-code-skills/skills/slidev .claude/skills/
```

### 2. Зарегистрируйте скилл

Добавьте в `.claude/settings.json`:

```json
{
  "skills": {
    "slidev": {
      "path": ".claude/skills/slidev/SKILL.md",
      "description": "Generate Slidev presentations from outlines",
      "trigger": "/slidev"
    }
  }
}
```

### 3. Используйте

```
/slidev my-outline.md
/slidev --preset corporate my-outline.md
/slidev --export png2pdf
/slidev --edit "сделай тёмную тему"
```

## Структура скилла

Каждый скилл имеет такую структуру директорий:

```
skills/
  my-skill/
    SKILL.md              # Главный файл инструкций (точка входа)
    references/           # Вспомогательные документы, гайды, примеры
      some-guide.md
      patterns.md
    assets/               # Шаблоны, демо-файлы
      demo-outline.md
```

**`SKILL.md`** — это ядро скилла. Он содержит полную процедуру, которой Claude следует. Думайте о нём как о детальном SOP (Standard Operating Procedure), написанном для ИИ, который умеет выполнять код, читать файлы и принимать решения.

## Как внести вклад

Мы приветствуем любой вклад! Вот как вы можете помочь:

### Добавить новый скилл

1. Форкните репозиторий
2. Создайте `skills/your-skill-name/SKILL.md`
3. Добавьте references и assets по необходимости
4. Тщательно протестируйте с Claude Code
5. Отправьте PR с:
   - Понятным описанием того, что делает скилл
   - Примерами использования и вывода
   - Списком зависимостей (npm-пакеты, Python-библиотеки и т.д.)

### Улучшить существующий скилл

Нашли баг? Знаете подход лучше? Скиллы — живые документы:

1. Форкните репозиторий
2. Отредактируйте `SKILL.md` или файлы в `references/`
3. Опишите в PR что и почему вы изменили
4. Бонус: приложите примеры до/после

### Критерии качества

Хороший скилл должен быть:

- **Конкретным** — решает определённую задачу, а не абстрактную
- **Самодостаточным** — все инструкции в одном месте, без неявных знаний
- **Проверяемым** — даёт наблюдаемый результат, который можно оценить
- **Документированным** — описывает подкоманды, опции и крайние случаи
- **Устойчивым** — обрабатывает ошибки, валидирует входные данные, имеет запасные стратегии

## Как написать свой скилл

### Начните просто

```markdown
# Мой скилл

Ты делаешь X, когда пользователь просит Y.

## Процедура

1. Прочитай входные данные
2. Сделай нужное
3. Выдай результат
```

### Потом итерируйте

Лучшие скиллы рождаются через реальное использование. Используйте паттерн `--learn`:

1. Напишите базовый скилл
2. Используйте его на реальных задачах
3. Отметьте, что ломается или даёт плохой результат
4. Добавьте правила для этих случаев
5. Повторите

### Ключевые принципы

- **Будьте явными** — Claude следует инструкциям буквально. "Сделай красиво" не работает; "Используй 3+ масштаба типографики, иерархию акцентного цвета с 3 уровнями прозрачности и чередуй плотные/дышащие слайды" — работает.
- **Включайте примеры** — Покажите точный код/вывод, который ожидаете, а не просто описания.
- **Добавляйте ограничители** — Помечайте критичные правила как `**CRITICAL**` или `MUST`. Claude уважает иерархию акцентов.
- **Используйте ссылки** — Разделяйте сложные скиллы на основной SKILL.md + reference-документы. Основной файл остаётся фокусированным.
- **Тестируйте крайние случаи** — Запустите скилл 10+ раз с разными вводными. Каждый сбой = новое правило.

## FAQ

**В: Чем скилл отличается от системного промпта или CLAUDE.md?**
Скиллы вызываются по требованию (через `/команду`), а CLAUDE.md загружается всегда. Используйте CLAUDE.md для общих конвенций проекта, скиллы — для конкретных рабочих процессов.

**В: Могут ли скиллы вызывать другие скиллы?**
Не напрямую, но скилл может дать Claude инструкцию прочитать файлы другого скилла.

**В: Какой максимальный размер скилла?**
Жёсткого лимита нет, но держите SKILL.md до ~4000 строк. Если больше — выносите в references.

**В: Работают ли скиллы с другими AI-инструментами для кода?**
Скиллы спроектированы для Claude Code, но концепция (структурированные markdown-инструкции) работает с любым ИИ, который читает файлы. Части, привязанные к инструментам (Bash, Read, Edit и т.д.), потребуют адаптации.
